## Introduction

**Apache DolphinScheduler**（海豚调度）是一个分布式、易扩展的**可视化 DAG（有向无环图）工作流任务调度系统**。它的核心目标是解决大数据处理、数据仓库建设以及日常复杂业务场景中的**任务编排与调度**问题。

#### 核心特点与功能：

1. **可视化 DAG 拖拽**：这是它最大的卖点。用户可以通过 Web 界面直接拖拽任务节点、连线来定义任务依赖关系，无需编写复杂的代码或 XML 配置文件，极大地降低了使用门槛。
2. **丰富的任务类型支持**：原生支持 Shell、Python、Spark、Flink、MapReduce、Hive SQL、DataX、SQOOP、Flink 等主流大数据和脚本组件。
3. **高可用与去中心化**：采用去中心化的多 Master 和多 Worker 架构，基于 [ZooKeeper]()（或 Registry Center）实现集群的高可用，避免单点故障。
4. **多租户系统**：支持 Linux 用户级别的多租户隔离，适合企业级多团队共用一个集群。
5. **强大的告警与容错机制**：支持邮件、企业微信、钉钉、WebHook 等多种告警方式；任务失败支持自动重试或手动恢复（支持从失败节点继续执行，无需重跑整个流程）。
6. **定时调度与依赖调度**：支持基于 Cron 表达式的定时调度，也支持跨工作流的自定义依赖调度。



## Architecture

DolphinScheduler 的架构可以清晰地划分为五个逻辑层：

1. **接入层**：包括 Web UI 和 API Server，负责接收用户请求、权限校验和指令下发。
2. **调度层（大脑）**：由分布式的 MasterServer 集群组成，负责 DAG（有向无环图）的解析、任务拆分、状态监控和调度分发。
3. **执行层（手脚）**：由分布式的 WorkerServer 集群组成，负责真正执行具体的任务（如跑 Shell、Spark、Flink 等）。
4. 协调与存储层（神经中枢与记忆）
   - **ZooKeeper / Registry Center**：负责集群节点管理、分布式锁、事件监听和任务队列协调。
   - **Database (MySQL/PostgreSQL)**：负责存储所有元数据（项目、工作流定义、租户）以及运行时的状态数据。
5. **辅助服务**：Alert Server 负责处理告警逻辑并发送通知（邮件、钉钉、企微等）





## 工作流

一个工作流从提交到执行完毕，其内部流转过程如下：

1. **指令生成**：用户通过 UI 点击“运行”，API Server 校验权限后，将一条 `Command` 写入 Database。
2. **Master 拉取**：MasterServer 集群会轮询 Database（或通过 ZK 事件触发），竞争拉取这条 Command。
3. **DAG 初始化**：抢到 Command 的 Master 将工作流实例状态初始化为“运行中”，并解析 DAG 图。
4. **任务入队**：Master 找出 DAG 中**没有前置依赖的根节点任务**，将其提交到 ZK 的分布式任务队列中。
5. **Worker 执行**：WorkerServer 监听到任务队列有新任务，拉取任务并执行。执行完成后，将状态结果写入 ZK（或更新到 DB）。
6. **状态流转**：Master 监听到某个任务执行成功，会更新 DAG 状态，并检查其**下游节点**。如果下游节点的所有前置依赖都已满足，则将这些下游任务继续提交到任务队列。
7. **循环与结束**：重复步骤 5 和 6，直到 DAG 中所有节点执行完毕，Master 将工作流状态标记为“成功”或“失败”
