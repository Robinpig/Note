## Introduction

| Parameter     | Default      | Description                                                                                                                                                                                    |
| --------------- | -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| initialSize   | 0            | 池启动时创建的初始连接数。<br/>Since: 1.2                                                                                                       |
| maxTotal      | 8            | 同时从该池分配的最大活跃连接数，负数表示无限制。                                                                      |
| maxIdle       | 8            | 池中可保持空闲的最大连接数，超过此数量的空闲连接将被释放，负数表示无限制。                                                               |
| minIdle       | 0            | 池中可保持空闲的最小连接数，低于此数量将创建新连接，零表示不创建。                                                                  |
| maxWaitMillis | indefinitely | 当没有可用连接时，池在抛出异常前等待连接返回的最大毫秒数，或 -1 表示无限等待。 |

NOTE: If maxIdle is set too low on heavily loaded systems it is possible you will see connections being closed and almost immediately new connections being opened.
This is a result of the active threads momentarily closing connections faster than they are opening them, causing the number of idle connections to rise above maxIdle.
The best value for maxIdle for heavily loaded system will vary but the default is a good starting point.

| Parameter                      | Default        | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ------------------------------ | -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| validationQuery                |                | 用于在将连接返回给调用者之前验证连接池中连接的 SQL 查询。如果指定，此查询**必须**是一个至少返回一行的 SQL SELECT 语句。如果未指定，将通过调用 isValid() 方法验证连接。                                                                                                                                                                                                                                                                                                                                                                                                                 |
| validationQueryTimeout         | no timeout     | 连接验证查询失败前的超时时间（秒）。如果设置为正值，该值将通过`Statement`的`setQueryTimeout`方法传递给驱动程序。                                                                                                                                                                                                                                                                                                                                                                                                                |
| testOnCreate                   | false          | 创建后是否验证对象的指示。如果对象验证失败，触发对象创建的借用尝试将失败。                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| testOnBorrow                   | true           | 从池中借用对象前是否验证的指示。如果对象验证失败，将从池中丢弃，并尝试借用另一个。                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| testOnReturn                   | false          | 对象返回池前是否验证的指示。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| testWhileIdle                  | false          | 空闲对象驱逐器（如有）是否验证对象的指示。如果对象验证失败，将从池中丢弃。                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| timeBetweenEvictionRunsMillis  | -1             | 空闲对象驱逐器线程运行间隔的毫秒数。如果为非正数，则不运行空闲对象驱逐器线程。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| numTestsPerEvictionRun         | 3              | 空闲对象驱逐器线程每次运行时检查的对象数量（如有）。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| minEvictableIdleTimeMillis     | 1000 * 60 * 30 | 对象在池中空闲的最小时间（毫秒），之后才有资格被空闲对象驱逐器（如有）驱逐。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| softMinEvictableIdleTimeMillis | -1             | 连接在池中空闲的最小时间（毫秒），之后才有资格被空闲连接驱逐器驱逐，额外条件是在池中至少保留"minIdle"个连接。当 minEvictableIdleTimeMillis 设置为正值时，空闲连接驱逐器首先检查 minEvictableIdleTimeMillis——即当驱逐器访问空闲连接时，首先将空闲时间与 minEvictableIdleTimeMillis 比较（不考虑池中空闲连接数），然后再与 softMinEvictableIdleTimeMillis 比较，同时考虑 minIdle 约束。 |
| maxConnLifetimeMillis          | -1             | 连接的最大生命周期（毫秒）。超过此时间后，连接将在下次激活、钝化或验证测试中失败。零或负数表示连接具有无限生命周期。                                                                                                                                                                                                                                                                                                                                                                                                   |
| logExpiredConnections          | true           | 标志，用于记录因超过 maxConnLifetimeMillis 而关闭连接的日志消息。设置为 false 可抑制默认开启的过期连接日志记录。                                                                                                                                                                                                                                                                                                                                                                                                                    |
| connectionInitSqls             | null           | 用于在首次创建物理连接时初始化连接的 SQL 语句集合。这些语句仅执行一次——当配置的连接工厂创建连接时。                                                                                                                                                                                                                                                                                                                                                                                                                |
| lifo                           | true           | true 表示 borrowObject 返回池中最近使用（"后进"）的连接（如果有空闲连接可用）。false 表示池表现为 FIFO 队列——连接按返回到池的顺序从空闲实例池中取出。                                                                                                                                                                                                                                                                                                                                         |
