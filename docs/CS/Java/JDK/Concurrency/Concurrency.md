## Introduction

[Threads](/docs/CS/Java/JDK/Concurrency/Thread.md) 是 Java 语言不可避免的特性，它们可以通过将复杂的异步代码转换为更简单的直线代码来简化复杂系统的开发。
此外，线程是利用多处理器系统计算能力的最简单方式。
而且，随着处理器数量的增加，有效利用并发将变得更加重要。

一些驱动因素促使了允许同时执行多个程序的操作系统的发展：

- Resource utilization（资源利用率）
- Fairness（公平性）
- Convenience（便利性）

线程有时被称为轻量级进程，大多数现代操作系统将线程（而非进程）作为基本的调度单元。
在没有显式协调的情况下，线程之间同时且异步地执行。
由于线程共享其所属进程的内存地址空间，同一进程内的所有线程都可以访问相同的变量并从同一堆中分配对象，
这使得比进程间机制更细粒度的数据共享成为可能。
但如果没有显式同步来协调对共享数据的访问，一个线程可能会修改另一个线程正在使用的变量，导致不可预测的结果。

**线程的优势**

1. 利用多个处理器
2. 建模简单性
3. 简化异步事件处理
4. 更响应的用户界面

**线程的风险**

1. 安全风险
2. 活跃性风险
3. 性能风险
